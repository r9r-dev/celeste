# Changelog
All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html),
and is generated by [Changie](https://github.com/miniscruff/changie).


## v2.1.0 - 2026-01-07
### Changed
* Improve error logging middleware to only log client/server errors (4xx, 5xx) and skip success responses (2xx) and redirects (3xx)

## v2.0.2 - 2026-01-07
### Fixed
* Clone reactive state arrays before passing to Chart.js to prevent Svelte 5 state_descriptors_fixed error

## v2.0.1 - 2026-01-07
### Fixed
* Fixed Dockerfile not using right bun.lock file

## v2.0.0 - 2026-01-06
### Changed
* Optimize CI pipeline: migrate from npm to bun, remove arm64 build, add Docker layer caching (12min -> ~1-2min)

## v1.0.0 - 2026-01-06
### Added
* Centralized version management with backend/internal/version package
* Version displayed in frontend header and /health endpoint
* Changie configuration for release management
### Changed
* WebSocket client renamed to websocket.svelte.ts for Svelte 5 runes convention
* API client uses dynamic URLs (window.location.origin) for production deployment
* Gin logger now only logs errors (non-2xx responses) to reduce noise
